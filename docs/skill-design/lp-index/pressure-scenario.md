# lp-index 压力场景（TDD）

## 失败模式

汇编学习路径索引 README 时常见的失败：

1. **目录树化**：README 只列文件名/标题，没有学习引导价值。
2. **无学习路径**：没有按依赖关系给出"先读 X 再读 Y"的阅读顺序，学习者不知从何读起。
3. **编造时间/状态**：元信息里的生成时间、文档完成状态凭空写，不查 `_meta/progress.json`。
4. **死链**：对选了但还没生成的文档给链接，或静默忽略未生成文档让学习者以为清单完整。
5. **缺项目速览**：没有一句话让学习者建立"这是什么项目"的认知就跳进功能清单。

## 验证方式

跑 lp-index 产出 `README.md`，对照 5 条铁律逐条核对：

- [ ] 开篇有**项目一句话简介**（从 project-knowledge/01 提炼）+ 链接到 01/02 背景。
- [ ] 每个已生成的功能文档有**「何时读」触发式描述**。
- [ ] 有**学习路径**章节：按依赖关系给出推荐阅读顺序（先读哪个、再读哪个），覆盖"从零到理解主线"≥1 条。
- [ ] 有**元信息**章节：从 `_meta/progress.json` 读取（项目名/生成时间/功能清单/选中项/完成状态），缺失字段显式标注，**禁止编造**。
- [ ] **区分已生成/未生成**文档（选了但没生成的标"待生成"，不给死链）。

## GREEN 规则

README 含可用学习路径（基于依赖字段排序：如"先读无依赖的 A，再读依赖 A 的 B"）；元信息时间来自 `_meta/progress.json` 不编造；死链 0 个（未生成文档显式标"待生成"）；5 条铁律全满足。满足即 GREEN。
