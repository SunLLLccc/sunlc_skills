# lp-prompt-gen 压力场景（TDD）

## 失败模式

为某个功能生成"教学提示词"时常见的失败：

1. **太泛**：提示词写成"请详细讲讲这个功能"，没给 doc 生成模型足够约束 → 产出跑偏或臆造。
2. **缺证据清单**：没列出要读的具体代码路径 → doc 生成模型不读真实代码就编，文档与代码脱节。
3. **缺学习目标**：没规定"读者读完后应能回答什么" → 文档没有聚焦点。
4. **缺文档结构/受众**：没规定 6 节结构、没声明受众（有经验但不熟本项目）→ 文档风格错位（讲通用语法、堆代码不讲 WHY）。
5. **不可复用重跑**：提示词混在对话里没存成文件，无法单独修改重跑。

## 验证方式

对 inventory.md 中选中的功能跑 lp-prompt-gen，每个产出 `features/prompts/{功能名}.md`，逐条核对：

- [ ] 含**必读证据**（≥1 条真实路径 + 说明），指向该功能核心代码。
- [ ] 含**学习目标**（读者读完后应能回答的具体问题）。
- [ ] 含**6 节文档结构**（概览/核心概念/如何工作/使用示例/与其它部分关系/延伸）。
- [ ] 含**受众声明**（有经验但不熟本项目）+ **撰写要求**（讲 WHY、引真实路径、不讲通用语法）。
- [ ] 存为独立文件，可单独修改重跑。

## GREEN 规则

每个提示词的"必读证据"列出的文件在项目中真实存在且指向该功能核心（非泛泛的整个模块目录）；结构完整（证据/学习目标/6 节结构/受众/撰写要求五要素齐全）；可复用重跑。满足即 GREEN。
