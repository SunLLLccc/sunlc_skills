# pm-conventions — 从代码实际推断的开发规范提取

## 定位

pm-conventions 是 project-mastery 学习管线的**波次 2** 规范提取 skill。它扫描项目源码与协作文档，从**代码实际**与**显式文档**推断开发规范，产出供后续维护与新成员 onboarding 使用的规范文档。

与凭印象列"业界最佳实践"的区别：本 skill 的每一条规范都必须能在代码或显式文档中找到证据，并标注置信度与来源类型，不臆造、不凑数。

## 输入

- `PROJECT_ROOT`：目标项目的根目录绝对路径
- `01-项目概览.md`、`02-技术栈与架构.md`：波次 1/2 的产出（作为上下文输入，提供项目类型、目录结构、技术栈、自封装框架位置）

## 产出

- `{PROJECT_ROOT}/docs/project-knowledge/03-开发规范.md`

## 核心原则：从代码实际推断，不臆造

每一条规范都必须满足以下三条铁律，否则禁止写入产出：

1. **有证据**：每条规范必须附【证据】字段，证据最小要求是 `文件路径 + 关键代码片段/模式描述`。只给文件名、不给片段或模式描述，不算合格证据。
2. **有置信度**：每条规范必须附【置信度】字段（高/中/低），分级标准见下。低置信度规范必须额外标注"仅个别样例，需进一步确认"。
3. **有来源类型**：每条规范必须附【来源类型】字段，区分规范是项目显式声明还是从代码反推的：
   - `显式文档`：来自 CONTRIBUTING.md / CLAUDE.md / AGENTS.md / README.md / .editorconfig / lint 配置等项目明文协作文档
   - `代码推断`：从源码一致模式反推（无显式文档声明）
   - `混合`：显式文档声明 + 代码实际一致

**禁止行为**：列出"Controller 层只做参数校验""Service 必须写单元测试"等通用最佳实践而未在代码中验证；编造项目并不存在的"CSS 方案""测试覆盖率要求"。

## 置信度分级标准

| 等级 | 标准 | 处理 |
|------|------|------|
| **高** | 多处（≥3 个文件/模块）代码模式一致，或有显式文档声明且代码与之一致 | 直接写入，作为规范陈述 |
| **中** | 多数样例一致但存在例外，或仅 2 个样例一致 | 写入并注明"多数一致，存在例外"或样例数 |
| **低** | 仅 1-2 个样例外推，无显式文档支撑 | **降级为"记录现有样例"**而非陈述规范，并显式标注"仅个别样例，需进一步确认" |

**关键规则**：低置信度不等于"凑成规范"。当某维度样例不足以支撑规范（如全项目只有 1-2 个测试类、只有个别组件用了某种样式方案），**禁止外推为规范**，正确做法是：记录"项目当前仅 N 个样例，做法是 X，置信度低，规范未成型"。

## 分析策略

从代码实际推断开发规范，覆盖 7 维度（目录结构/命名/代码封装/异常处理/注释/提交规范/...）。每条规范附代码证据 + 置信度（已验证=代码中明确一致模式；推断=零散迹象需人工确认）。

详细 7 维度推断方法、显式文档来源清点、边界与豁免见 `references/conventions-rules.md`（执行时按需读取）。

## 输入输出预算（小模型纪律）

- **单步单产物**：本次只产 `03-开发规范.md`（一篇），不并行产其它。
- **读预算**：先读显式协作文档（CONTRIBUTING/CLAUDE/AGENTS/README/lint 等，≤8 份）+ 抽样代表性源文件 ≤15 个，单文件 ≤500 行；禁扫 node_modules/dist/target/build/.git/vendor/__pycache__；限深 ≤3 层。
- **写预算**：03 默认 8000-12000 字；7 维度样例不足的降级为"记录现有样例（低置信度）"，不外推凑数。
- **证据先行**：每条规范附【证据】（文件+片段）+【置信度】+【来源类型】；无证据不入册。

## 执行检查清单

执行 pm-conventions 时，按以下顺序完成：

1. [ ] 读取 `01-项目概览.md`、`02-技术栈与架构.md` 作为上下文输入
2. [ ] 扫描显式协作文档（CONTRIBUTING/CLAUDE/AGENTS/README/lint 配置等），记录路径与要点
3. [ ] 逐维度分析 7 个规范维度，每个维度抽样代表性文件
4. [ ] 每条规范填写【证据】【置信度】【来源类型】三字段
5. [ ] 不适用维度注明"未发现/不适用"及原因
6. [ ] 样本不足的维度降级为"记录现有样例 + 低置信度"，不外推
7. [ ] 按模板格式写入 `03-开发规范.md`
8. [ ] 自检：每条规范有三字段？证据含文件+片段？低置信度已标注？不适用维度已注明？无臆造规范？
